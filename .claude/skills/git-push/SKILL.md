# git-push

작업 변경사항을 검토하고 `add` → `commit` → `push` 까지 안전하게 진행합니다.

## 진행 순서

1. **상태 확인**
   - `git status` 와 `git diff --stat` 으로 무엇이 바뀌었는지 파악한다.
   - `git remote -v`, `git branch -vv` 로 원격/추적 브랜치를 확인한다.

2. **비밀 정보 점검 (필수)**
   - 스테이징 대상에 비밀 정보가 없는지 반드시 확인한다.
   - 다음 같은 파일/내용이 포함되면 **커밋하지 말고 사용자에게 알린다**:
     - `key.md`, `.env`, `.env.local`, `*.pem`, `id_rsa`, credential/secret 파일
     - API 키, 비밀번호, JWT(`eyJ...`), service_role 키 등이 들어간 일반 파일
   - 의심되는 파일은 `git rm --cached <file>` 로 제외하고 `.gitignore` 에 추가한 뒤 알린다.
   - 이 프로젝트에서 `key.md` 는 절대 커밋 금지 (실제 카카오/Supabase 키 보관 파일).

3. **add**
   - 의도한 파일만 스테이징한다. 보통 `git add -A` 로 전체 추가하되, 2번에서 걸러진 파일은 제외 상태를 유지한다.

4. **commit**
   - 사용자가 커밋 메시지를 주면 그대로 사용한다.
   - 메시지가 없으면 변경 내용을 보고 간결한 한국어 커밋 메시지를 직접 작성한다 (예: `식당 추가 폼 유효성 검사 추가`).
   - 명령: `git commit -m "<메시지>"`

5. **push**
   - 현재 브랜치를 원격으로 푸시한다: `git push`
   - 업스트림이 없으면: `git push -u origin <현재브랜치>`

6. **결과 보고**
   - 푸시된 커밋 해시와 원격 브랜치를 알려준다.
   - 실패 시 에러 원문을 그대로 보여주고 원인을 설명한다 (인증 실패, 충돌 등).

## 주의사항

- `--no-verify` 로 훅을 건너뛰지 않는다.
- `--force` 푸시는 사용자가 명시적으로 요청할 때만 한다.
- main 브랜치 보호가 필요한 상황이면 사용자에게 먼저 확인한다.
- PowerShell 환경에서는 명령 연쇄에 `&&` 대신 `;` 와 `if ($?) { ... }` 를 쓴다.
